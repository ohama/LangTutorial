## v5.0 Requirements

타입 에러 진단 시스템. Algorithm W의 에러 위치/원인을 정확히 표현.

### Span/Range

- [x] **SPAN-01**: Span 타입 정의 (파일, 시작/끝 라인·컬럼)
- [x] **SPAN-02**: Expr 노드에 span 필드 추가
- [x] **SPAN-03**: Lexer에서 위치 정보 생성
- [x] **SPAN-04**: Parser에서 AST에 span 전달

### Diagnostic

- [x] **DIAG-01**: Diagnostic 타입 정의 (code, message, primarySpan, secondarySpans, notes, hint)
- [x] **DIAG-02**: TypeError 타입 정의 (kind, span, expected, actual, term, contextStack, trace)
- [x] **DIAG-03**: TypeError kind 정의 (UnifyMismatch, OccursCheck, UnboundVar, NotAFunction 등)
- [x] **DIAG-04**: TypeError → Diagnostic 변환 함수

### Context Stack

- [x] **CTX-01**: InferContext 타입 정의 (InIfCond, InIfThen, InIfElse, InAppFun, InAppArg, InLetRhs 등)
- [x] **CTX-02**: 타입 추론 시 context stack 관리
- [x] **CTX-03**: 에러 발생 시 context stack 포함

### Unification Trace

- [x] **TRACE-01**: UnifyPath 타입 정의 (AtFunctionReturn, AtTupleIndex, AtListElement 등)
- [x] **TRACE-02**: unify 실패 시 충돌 경로 기록
- [x] **TRACE-03**: Trace 정보를 TypeError에 포함

### Blame Assignment

- [x] **BLAME-01**: Primary span 선택 규칙 구현 (가장 직접적인 원인)
- [x] **BLAME-02**: Secondary span 선택 규칙 구현 (관련 expr)
- [x] **BLAME-03**: 가장 안쪽 expr 우선 규칙

### Error Output

- [x] **OUT-01**: 에러 코드 체계 정의 (E0301 등)
- [x] **OUT-02**: 에러 메시지 포맷 (위치, 기대/실제 타입, context 요약, 힌트)
- [x] **OUT-03**: 타입 pretty-printer 개선 (타입 변수 a,b,c 정규화)
- [x] **OUT-04**: CLI에서 새 에러 포맷 출력

### Testing

- [x] **TEST-01**: if 조건이 int인 경우 테스트
- [x] **TEST-02**: 함수가 아닌 값 호출 테스트
- [x] **TEST-03**: 인자 타입 불일치 테스트
- [x] **TEST-04**: let rhs 에러 테스트
- [x] **TEST-05**: occurs check 테스트
- [x] **TEST-06**: Diagnostic 골든 테스트 프레임워크
